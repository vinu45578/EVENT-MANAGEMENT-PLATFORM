### Purpose
The purpose of this Software Requirements Specification (SRS) document is to describe the functional and non-functional requirements of the **Event Management Platform**. This document serves as a reference for developers, testers, project stakeholders, and evaluators to clearly understand the system behavior, features, and constraints.

### Intended Audience
This document is intended for:
- Project team members involved in design, development, and testing
- Faculty and evaluators assessing the project
- Future developers and maintainers of the system

### Product Overview
The Event Management Platform is a web-based application designed to centralize the management of college events and volunteering activities. The system enables administrators to create and manage events, while students can view event details, register for participation, and enroll as volunteers. It improves communication, tracks attendance, maintains participation records, and supports recognition through certificates and credits.

### Document Organization
This document is organized into sections that describe the system overview, functional requirements, non-functional requirements, constraints, and assumptions. Together, these sections provide a comprehensive understanding of the Event Management Platform and its intended functionality.
