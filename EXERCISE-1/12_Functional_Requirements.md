## Functional Requirements

Functional requirements describe what the **Event Management Platform** is expected to do.

### FR1: User Authentication
The system shall allow users to log in based on their role (Student, Event Organizer, or Administrator).

### FR2: Event Creation
The system shall allow administrators to create events by providing details such as event name, date, time, venue, and description.

### FR3: Event Management
The system shall allow administrators to update, edit, or delete event information.

### FR4: View Events
The system shall allow students to view a list of upcoming and ongoing events.

### FR5: Event Registration
The system shall allow students to register for events.

### FR6: Volunteer Enrollment
The system shall allow students to enroll as volunteers for events with specified roles.

### FR7: Attendance Tracking
The system shall allow organizers to record and manage attendance of participants and volunteers.

### FR8: Notifications and Updates
The system shall notify students about event registrations, updates, or changes.

### FR9: Participation Records
The system shall maintain records of student participation and volunteering history.

### FR10: Reports and Summaries
The system shall generate basic reports related to event participation and attendance.

These functional requirements define the core features and expected behavior of the Event Management Platform.
