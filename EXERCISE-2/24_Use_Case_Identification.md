## Use Case Identification

Use cases describe the interactions between the actors and the **Event Management Platform**. Each use case represents a specific functionality provided by the system.

### Identified Use Cases

#### Login
Allows students, event organizers, and administrators to log in to the system based on their roles.

#### View Events
Allows students to view upcoming and ongoing events along with event details.

#### Register for Event
Allows students to register for events they wish to participate in.

#### Volunteer Enrollment
Allows students to enroll as volunteers for events by selecting available roles.

#### Create Event
Allows event organizers or administrators to create new events by providing event details.

#### Manage Events
Allows administrators to update, modify, or delete existing events.

#### Track Attendance
Allows organizers to record and manage attendance of participants and volunteers.

#### View Participation Records
Allows students to view their event participation and volunteering history.

These use cases define the core functional interactions of the **Event Management Platform** and form the basis for system design and implementation.
