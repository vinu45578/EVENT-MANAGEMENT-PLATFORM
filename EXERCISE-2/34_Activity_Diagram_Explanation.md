## Activity Diagram Explanation

The activity diagram represents the flow of actions performed in the **Event Management Platform**.

### Explanation

1. The process starts when the user logs into the system.
2. After login, the user can choose different actions based on their role:
   - Students can view events, register for events, or enroll as volunteers.
   - Administrators can create and manage events.
3. When a student registers or enrolls as a volunteer, the system validates the details.
4. The system stores the registration or volunteer information.
5. The system updates participation records and reflects the changes on the dashboard.
6. Users can view participation or attendance summaries.
7. The process ends when the user logs out of the system.

This activity diagram helps in understanding the overall workflow and sequence of activities within the **Event Management Platform**, providing clarity on how users interact with the system from start to finish.
